# 系统完备性设计（手感 / 界面 / 工程化）· 明朝抗倭

> 本文补齐 `09-architecture.md` 之外、主流割草（吸血鬼幸存者 / Brotato / Halls of Torment / Soulstone Survivors）不可或缺、且**越晚加越贵**的系统。延续全局原则：**逻辑与表现解耦、数据驱动、原语只增不改、接口提前预留**。
>
> 分档：**A 手感必需（MVP 内）** / **B 现在留接口（后期改贵）** / **C MVP 后**。所有"留接口"项都在 M0~M8 期间按本文形状预埋空实现，避免返工。

---

## A 档 · 手感必需（MVP 内实现）

### A1. 打击感（Juice）系统
割草"爽感"的核心。逻辑只发事件，表现层（二之三）+ 专用反馈系统消费：
- **击退 Knockback**：命中给目标一个朝远离来源方向的冲量；做成 `KnockbackSpec{ force, decay }`，由 `MovementSystem` 衰减积分。Boss/精英有"抗性"系数。
- **受击僵直 Hitstun**：短暂打断敌人移动/AI（`Stunned{ remainS }` 组件），与"震慑"异常区分（僵直是命中反馈，震慑是异常状态）。
- **顿帧 Hit-stop**：命中瞬间全局时间缩放到 ~0.0（持续数十 ms）再恢复，强化打击；走 A5 的 `timeScale`，仅大命中/暴击/爆发触发，带 ICD 防频闪。
- **受击闪白 / 死亡反馈**：`PresentationSystem` 在 onTakeDamage 给 Sprite 叠白色 tint 数帧；死亡播碎裂/血雾特效（对象池）。
- **伤害飘字**：池化文本，暴击/异常/治疗分色分号；可在设置中关闭（见 C 档无障碍）。
> 全部参数化：每个绝技/敌人可在数据里覆盖击退力度、是否触发顿帧、闪白颜色等。

### A2. 相机系统
- **跟随 + 死区（deadzone）**：玩家在死区内移动相机不动，出死区才平滑跟随。
- **震屏**：消费 `screenShake` 反馈事件，trauma 衰减模型（避免持续抖动）。
- **缩放**：基础缩放 + 关键时刻（Boss 入场/觉醒）拉近；分辨率自适应（配合 Phaser Scale Manager）。
- **边界**：MVP 采用**有界竞技场**（江南水乡一张图），相机夹取在地图边界内；预留"无限滚动"开关供未来地图用。

### A3. HUD / UI 数据绑定架构
UI 不直接读写游戏逻辑，**单向订阅**：ECS/core 状态变化 → 发 UI 事件 / 暴露只读快照 → UI 层渲染。
- **绑定方式**：轻量响应式（一个 `UiState` 只读视图 + 事件通知），UI 组件订阅所需字段；逻辑层不引用任何 UI。
- **MVP HUD**：血条、经验条/等级、关卡计时、击杀数、绝技 CD 图标组、当前异常/增益小图标、Boss 血条、获得拾取提示。
- **界面层**：用 Phaser 容器或叠加 DOM（二选一，M0 定）；文本全部走 i18n key（见 B1）。
- 升级三选一、暂停菜单、结算页都是 UI 场景，消费同一只读快照。

### A4. 音频系统
- **总线分层**：`master / bgm / sfx / ui` 音量总线，设置可独立调节并持久化（经 StoragePort）。
- **SFX**：`PresentationSpec.sfx` 触发；同音效并发上限 + 短 ICD 防"音爆"（割草同帧上百命中）。
- **BGM**：场景切换淡入淡出；Boss 战切轨；可选战斗强度 ducking。
- 资源走 AssetRegistry；web/桌面统一接口。

### A5. 暂停与时间缩放（timeScale）
- **统一 `timeScale`**：定步长循环的累加器步进乘以 `timeScale`。`0`=暂停、`1`=正常、`0~1`=慢动作、`>1`=调试加速。
- **暂停语义**：三选一抽卡 / 暂停菜单时 `timeScale=0`（逻辑冻结，UI 仍响应）；Boss 入场短暂慢放。
- **防 spiral-of-death**：每渲染帧最多执行 N 个逻辑 substep（如 5），超出丢弃累加，保证卡顿不雪崩。
- 顿帧（A1）= 极短的 `timeScale≈0` 脉冲。

### A6. 敌人 AI 多样性 + 防堆叠
- **行为原语（可复用，类比轨迹原语）**：`chase`（追玩家）/ `charger`（蓄力冲刺）/ `ranged`（保持距离+射击）/ `exploder`（接近自爆）/ `shielded`（正面减伤需绕后）/ `summoner`。存 `AIBehavior` 组件，`EnemyAISystem` 按 kind 派发纯函数。
- **防堆叠 separation**：转向时加入"与邻近同类的分离力"（boids separation），避免几百怪叠成一个点；用 A4 的空间网格查邻居，控制半径与权重。
- **官位词缀**（00 文档）：精英在行为上叠加光环/词缀，组合式装配，不新增类。

## B 档 · 现在留接口（架构便宜、后期改贵）

### B1. i18n 文本外置（强烈建议从 M0 做）
- **现状风险**：题材化中文（绝技名/描述）写死在 `config/`。Steam 全球化与 MOD 本地化要求 key 化，**早做近乎零成本，晚做要回头翻所有内容**。
- **方案**：配置里只存 `nameKey/descKey`，文案集中在 `locales/zh-CN.ts`（及未来 en-US…）；UI/卡牌渲染时按当前语言查表。MOD 可随包附带自己的 locale 片段。
- 描述支持插值（`{damage}`/`{stacks}`）以便数值随等级变化的动态文案。

### B2. 统一输入抽象（InputSystem 之上）
- 把"按键/手柄/触控"映射到**抽象意图**（`MoveDir / CastAwakening / Pause / Confirm / Reroll`），游戏逻辑只消费意图。
- 一份 `InputMap` 配置（可重绑定），实现：键鼠（web/桌面）、手柄（Steam）、虚拟摇杆（未来移动端）。
- 与 09 的 `InputSystem`（写 Intent 组件）衔接：输入抽象层产出意图 → InputSystem 落到组件。

### B3. 种子化对局 + 多 RNG 流
- 每局一个 `runSeed`；从中派生**独立子流**：`combat`（暴击/概率触发）、`draft`（抽卡）、`spawn`（刷怪）、`loot`（掉落）。互不串味，便于复现单一系统。
- 全部随机经注入 `Rng`（09 已有），不直接用 `Math.random`。
- 收益：bug 复现、回放（C 档）、每日挑战/种子分享（C 档）、平衡测试。

### B4. 伤害来源归因（attribution）
- 结算时给每次伤害打标 `sourceCardId / sourceAbilityId`；按来源累加到本局统计。
- 用途：玩家**结算面板伤害构成**（C5）、你**调数值**时一眼看出谁过强、UI 高亮"这张卡贡献了 X%"。
- 实现：`DamageInfo` 增 `attribution` 字段，`DamageSystem` 汇总进 `RunStats`。

### B5. 抽卡规则增强 + 武器进化（声明式）
扩展 09 第九节抽卡 + 10 的 `CardDef`：
- **前置依赖 `requires`**：某增益需先拥有某卡才入池。
- **互斥 `excludes`**：互斥卡不同时出现。
- **构筑加权 `weightWhen`**：按已有构筑动态调权（强化你正在走的流派）。
- **武器/绝技进化 `evolution`**：声明"满级 + 持有指定被动 → 质变为新形态"（吸血鬼幸存者式进化）。做成数据规则 `{ requires:[passiveId], atMaxLevel:true, into:newCardId }`，由抽卡/检测系统在满足时提供"进化"选项。

### B6. 难度缩放 + 诅咒/挑战修饰符
- **时间缩放曲线**：敌人 HP/伤害/密度随关卡时间按曲线提升（数据表，非硬编码）。
- **诅咒 / 挑战修饰符（run modifiers）**：全局修饰符（如"敌人+50%移速换+30%经验"），**复用卡引擎修饰符 + 刷怪参数覆盖**，挂在 `RunConfig` 上。供未来"诅咒层数 / 进阶难度 / 每日挑战"使用。

## C 档 · MVP 后路线（先留痕，不实现）

- **C1 结算统计面板**：DPS、伤害构成（依赖 B4）、击杀数、存活时长、构筑总览、获得卡牌时间线。
- **C2 设置项**：画质档位、音量（A4）、按键重绑（B2）、语言（B1）；**无障碍**：震屏开关、顿帧开关、伤害飘字开关、色盲friendly 异常配色。
- **C3 遥测（opt-in）**：匿名平衡数据（各卡选取率/胜率/伤害占比）上报，仅在用户同意下开启；用于持续调平衡。
- **C4 回放 / 录像**：基于 B3 种子 + 输入序列做确定性回放（world 纯数据，天然支持）；用于 bug 复现与精彩分享。
- **C5 宝箱 / 地图事件**：宝箱=多重升级（吸血鬼幸存者式）、随机地图事件（增援/资源点/精英刷新）。
- **C6 存档防篡改**：本地存档签名/校验（为 Steam 排行榜/成就铺垫）。
- **C7 每日挑战 / 种子分享**：固定种子全球同图竞速（依赖 B3）。
- **C8 地形与障碍**：可破坏物、环境危害、导航网格（若地图复杂度提升）。
