# M1 战斗核心 — 运行时测试报告 (PR #4)

测试方式：本地 `vite preview`（http://localhost:4173，dist 含修复后透明精灵）通过浏览器实操 T1–T5，全程录屏。

## 结论

本次修复 + 测试发现并修复了一个真实缺陷（重开键 R 失灵），修复后 5 项全部通过。

## 测试发现（先看这里）

- **[已修复] 重开键 R 失灵**：原实现用 `Phaser.Input.Keyboard.JustDown(addKey('R'))` 轮询，在死亡覆盖层下实体按键 R 连续 3 次均无法重开（合成事件可触发，说明 `restart()` 逻辑本身没问题，是按键检测脆弱）。已改为 `keyboard.on('keydown-R', …)` 事件监听，实体 R 键重开恢复正常。

## 测试结果

- T1 透明精灵渲染：**通过** — 刀牌手无白色方块，水乡地表干净。
- T2 自动攻击 + 击杀计数：**通过** — 戚家刀扇形横扫自动击杀倭寇，击杀从 0 递增至 50。
- T3 接触掉血 + 游戏结束：**通过** — 倭寇接触玩家血条耗尽至 0，触发"阵亡"覆盖层。
- T4 相机跟随：**通过** — 持续移动时水乡地表滚动，玩家保持视口中心死区内。
- T5 死亡 + 实体 R 重开：**通过（修复后）** — 覆盖层显示"阵亡 / 按 R 重新开始"，按 R 后血满、击杀归零、计时重置。

> 注：T3 的"相机震屏"在静态截图中不易体现，请在录屏中观感确认；接触掉血以血条耗尽至 0 验证。
